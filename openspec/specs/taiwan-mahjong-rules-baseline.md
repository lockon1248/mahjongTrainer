# 台灣 16 張麻將規則 baseline

本文件定義目前 AI 與規則引擎採用的台灣 16 張麻將 baseline。
若本文件未特別說明，先以常見台灣民間桌規理解；若遇到不同桌規，必須透過規則設定覆蓋，不得直接寫死於程式。

## 基本桌規

- 玩家數：4 人
- 麻將總數：144 張
  - 萬子 36 張
  - 筒子 36 張
  - 索子 36 張
  - 風牌 16 張
  - 三元牌 12 張
  - 花牌 8 張
- 閒家起手 16 張
- 莊家起手 17 張
- 莊家先出牌
- 每回合摸 1 打 1
- 胡牌後手牌共 17 張
- 牌摸完仍無人胡牌時流局

## 合法動作規則

- 摸
  - 輪到自己回合時，從牌牆摸 1 張牌。
- 打
  - 摸牌後或整理既有手牌後，打出 1 張牌。
- 吃
  - 只能吃上家打出的牌。
  - 吃牌後形成順子，例如 `123`、`234`、`789`。
- 碰
  - 任一家打出的牌皆可碰。
  - 碰牌者手中必須已有兩張相同牌。
- 明槓
  - 以他家打出的牌完成槓子。
  - 槓後須補牌。
- 暗槓
  - 以自己手中的四張相同牌組成槓子。
  - 槓後須補牌。
- 加槓
  - 已有碰子時，再以第 4 張相同牌加成槓子。
  - 槓後須補牌。
- 胡
  - 取得合法胡牌結構時可宣告胡牌。
- 過
  - 在可宣告吃、碰、槓、胡時選擇放棄該次宣告。

### 花牌補牌

- 花牌共 8 張：春、夏、秋、冬、梅、蘭、竹、菊。
- 摸到花牌後必須立即亮出，並從牌尾補牌。
- 若補到花牌，必須持續補牌直到補到非花牌。

### 宣告優先序

- 多人同時宣告時，baseline 先採 `胡 > 槓 > 碰 > 吃`。
- 若後續桌規要求不同優先序，必須由規則設定覆蓋，不得散落在 UI 或 store。

## 胡牌成立條件

- 台灣 16 張標準胡牌結構為：
  - 五組面子
  - 一組將眼
- 可用結構示意如下：

```text
AAA
ABC
ABC
AAA
ABC
AA
```

- 順子、刻子、槓子、將皆可構成合法胡牌牌型。
- 聽牌表示只差一張即可胡牌，常見型態可包含單吊、邊張、嵌張、雙碰、多面聽。

## 台型清單

本 baseline 的 scoring 主線改採「先固定權威工作清單，再逐批定案並落地」：

### 第一批必做台型主線

- 莊家胡
- 自摸
- 最低胡牌台數門檻
- 天胡
- 大三元
- 小三元

### 第一批已定值規則

- 最低胡牌台數：`0 台起胡`
- 天胡：啟用，`24 台`
- 大三元：啟用，`8 台`
- 小三元：啟用，`4 台`

### 第二批待擴充台型主線

- 花牌計分
- 風牌相關台型
- 門清相關台型
- 對對胡 / 混一色 / 清一色等一般進階台型
- 其他特殊胡型

### 權威規則落地原則

- 每一個台型在進入程式邏輯前，必須先在本文件或後續權威 spec 中補上：
  - 台型名稱
  - 是否啟用
  - 台數值
  - 是否可與其他台型疊加
  - 若屬特殊胡型，需標示成立條件與是否覆蓋一般台型
- 未完成上述資料前，不得直接把台型數值寫進 scoring code。
- 若某台型僅列為主線工作項目而尚未定值，程式只能保留開關或待實作入口，不得自行猜數值。

## 結算規則

- 自摸
  - 摸進最後所需牌張並完成胡牌時，視為自摸。
  - baseline 先記錄常見桌規為自摸三家付。
  - 實際支付金額與是否另有包牌機制，必須由桌規設定決定。
- 放槍
  - 玩家打出他人可胡之牌張時，視為放槍。
  - baseline 先記錄常見桌規為放槍者單獨支付。
  - 實際支付金額與是否一炮多響，必須由桌規設定決定。
- 莊家
  - 莊家胡牌時可連莊。
  - 閒家胡牌時，下家坐莊。

## 待確認／爭議規則

以下項目目前不得直接寫死；若未有明確設定或後續 spec，實作時必須視為待確認或待定值：

| 項目 | 狀態 | 備註 |
| 台型完整清單 | 進行中 | 已先固定第一批與第二批工作清單，細項仍需逐批定值 |
| 特殊胡型是否採用 | 進行中 | 第一批至少含天胡、大三元、小三元；其餘特殊胡待後續批次 |
| 搶槓規則 | 待確認 | 是否允許、何時可搶槓 |
| 是否封頂 | 待定值 | 是否有上限與上限值需明定 |
| 過水限制 | 待確認 | 過手可否再胡或再宣告 |
| 流局是否查聽 | 待確認 | 流局後是否需要依聽牌狀態處理 |
| 花牌是否直接計分 | 進行中 | 已列入第二批 scoring 主線，計分方式待定值 |
| 最低胡牌台數 | 已定值 | 採 `0 台起胡` |

## 可配置規則

以下規則不得硬編碼，必須以 `Rule Config` 或同等結構表示：

| 規則項目 | 預設 baseline | 備註 |
| --- | --- | --- |
| 是否可吃 | 是 | baseline 採可吃上家 |
| 是否可槓上家 | 待確認 | 不同桌規差異大 |
| 自摸是否三家包 | 是 | 金額與附加條件不可寫死 |
| 放槍是否包牌 | 否／待確認 | 先保留為設定項 |
| 花牌計分 | 待定值 | 需由權威 scoring 規則定義 |
| 最低胡牌台數 | `0 台起胡` | 牌型成立即可胡牌 |
| 是否天地胡 | 待確認 | 若採用，需拆分天胡 / 地胡定義與台數 |
| 是否天胡 | 啟用，24 台 | 只屬於莊家，配牌後莊家尚未打出第一張牌前已成和 |
| 是否大三元 | 啟用，8 台 | 中、發、白三組刻子 |
| 是否小三元 | 啟用，4 台 | 中、發、白其中兩組刻子加一組對子 |
| 是否七搶一 | 待確認 | 特殊胡型 |
| 是否八仙過海 | 待確認 | 特殊胡型 |
| 是否允許搶槓 | 待確認 | 與宣告流程相關 |
| 是否過水限制 | 待確認 | 與宣告窗口相關 |
| 是否一炮多響 | 待確認 | 與結算流程相關 |
| 連莊規則 | 莊家胡牌連莊 | 其他條件待後續明定 |
| 拉莊規則 | 待確認 | 不在本 baseline 內寫死 |
| 流局是否查聽 | 待確認 | 不在本 baseline 內寫死 |
| 花牌補牌方式 | 牌尾補牌 | 補到花需連續補牌 |

## 測試案例對照

- `FLOWER-REPLACE-*`：驗證摸花立即亮出、尾端補牌、補到花繼續補
- `CLAIM-PRIORITY-*`：驗證多人同時宣告時依 `胡 > 槓 > 碰 > 吃` 處理
- `WIN-STANDARD-*`：驗證五組面子加一組將眼的標準胡牌結構
- `WIN-MELD-*`：驗證副露後仍可成立標準胡牌
- `DRAW-*`：驗證牌摸完無人胡牌時流局
- `SCORE-*`：驗證自摸與放槍的基礎支付責任來自規則設定
- `SCORE-DISCARD-WIN-*`：驗證榮和結果也必須經過 scoring evaluation 與 settlement
- `SCORE-MINIMUM-TAI-*`：驗證最低胡牌台數門檻會影響和牌是否可成立
- `PATTERN-HEAVEN-WIN-*`：驗證天胡特殊胡型的成立條件與結果摘要
- `PATTERN-BIG-THREE-DRAGONS-*`：驗證大三元特殊胡型的成立條件與結果摘要
- `PATTERN-LITTLE-THREE-DRAGONS-*`：驗證小三元特殊胡型的成立條件與結果摘要
